🛠️ Tecnologie usate

Riepilogo di tutto quello che abbiamo usato per costruire il sito.

📄
Markup

HTML5

La struttura di tutte le pagine è scritta in HTML5. Abbiamo cercato di usare i tag semantici corretti (nav, main, footer, section) invece di mettere tutto in div generiche.

🎨
Stile

CSS3

Tutto lo stile è in un unico file style.css. Abbiamo usato variabili CSS, Flexbox e CSS Grid. Nessun framework CSS come Bootstrap — volevamo scrivere tutto da soli per imparare.

⚡
Logica

JavaScript

Un file script.js condiviso tra tutte le pagine. Le parti più semplici le abbiamo scritte da soli, quelle più complesse (filtro ricerca, FAQ, contatore) con l'aiuto dell'IA e documentando nel codice.

🔤
Font

Google Fonts

Usiamo due font: Cabinet Grotesk per il testo normale e DM Mono per il codice, le label e i badge. Entrambi sono gratuiti e si caricano via @import CSS.

🤖
Aiuto

Intelligenza Artificiale

Alcune parti del JavaScript sono state scritte con l'aiuto di ChatGPT e Claude. Le abbiamo segnalate tutte con il commento [AIUTO IA] nel file script.js, spiegando cosa fa il codice e che bug avevamo prima.

🐙
Hosting

GitHub Pages

Il sito è pubblicato gratuitamente su GitHub Pages. Il repository è pubblico e contiene tutti i file sorgente del progetto.

❓ Domande frequenti

Risposte alle domande che il professore potrebbe farci durante la presentazione.

Perché avete scelto questo argomento?
Abbiamo scelto le risorse per lo sviluppo web perché è l'argomento del corso e perché quando abbiamo iniziato a studiare ci siamo resi conto che c'erano tantissimi siti utili ma nessuno li raccoglieva in modo organizzato. Abbiamo pensato che poteva essere utile anche per i compagni di classe.
Avete usato framework CSS come Bootstrap o Tailwind?
No. Abbiamo scelto di scrivere tutto il CSS da zero perché volevamo imparare e capire come funzionano le proprietà CSS senza dipendere da classi predefinite di un framework. Abbiamo usato solo i font di Google Fonts, che si importano con una riga di CSS.
Come avete usato l'intelligenza artificiale?
Abbiamo usato l'IA principalmente per le parti di JavaScript più complesse: il filtro di ricerca delle card, l'accordion delle FAQ (quella che state leggendo) e l'animazione dei numeri nella homepage. Tutte queste parti sono segnalate nel codice con il commento [AIUTO IA] e abbiamo scritto una spiegazione di come funzionano e che problemi avevamo prima di chiedere aiuto.
Il sito è responsive? Funziona su smartphone?
Sì. Abbiamo usato i media query CSS per adattare il layout agli schermi piccoli. Su mobile la navbar mostra il menu hamburger, la griglia delle card passa a una sola colonna e il footer si dispone in verticale. Abbiamo testato su Chrome con gli strumenti per sviluppatori (modalità dispositivo mobile).
Quante ore ci avete lavorato?
Circa 15-20 ore in totale. Abbiamo iniziato dalla struttura HTML e dal CSS di base, poi abbiamo aggiunto i contenuti di ogni sezione, e infine abbiamo aggiunto il JavaScript. La parte che ha richiesto più tempo è stata il CSS della navbar e la parte responsive del sito.
Cosa cambiereste se aveste più tempo?
Aggiungeremmo la possibilità di filtrare le risorse per tipo (Gratis, A pagamento, Open Source) con dei bottoni, non solo con la barra di ricerca. Aggiungeremmo anche una sezione dove gli utenti possono suggerire nuove risorse tramite un form. E forse una dark mode completa invece del toggle parziale che c'è adesso.
